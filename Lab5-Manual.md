# Лабораторна робота № 5

## Розробка серверної частини вебсайту

### Хід виконання

Лабораторна робота № 5 є логічним продовженням лабораторної роботи № 4. Результатом її виконання є розробка back-end частини (API) з інформацією про країни.

Тобто, інформація про країни буде надходити не з сайту `https://restcountries.com`, а з розробленої back-end частини (рис. 1).

**(Рисунок 1 - Архітектура вебсайту)**

```
                     +----------------------+
                     |      Веббраузер      |
                     |      (Browser)       |
                     +-----------+----------+
                                 |
                                 |  HTTP-запит / HTML-відповідь
                                 |
                     +-----------v----------+
                     | Front-end частина    |
                     |     (Лаб. 4)         |
                     +-----------+----------+
                                 |
                                 |  HTTP-запит / JSON-відповідь
                                 |
                     +-----------v----------+
                     | Back-end частина     |
                     |      (Лаб. 5)        |
                     +----------------------+
```

---

### Для виконання завдання необхідно:

**1. Створити на сервері обробку URL-адрес, вказаних у таблиці:**

| URL-адреса | Призначення |
| :--- | :--- |
| `http://localhost/mycountries/v1.1/getcountrieslist` | Повертає перелік країн з ідентифікаторами. Використовується для формування випадаючого списку. |
| `http://localhost/mycountries/v1.1/getcountry/:countryname` | Повертає інформацію про обрану країну на основі отриманого ідентифікатора (аліаса) `:countryname`.<br>_Приклад: `.../getcountry/ukraine`_<br>Використовується при виборі нової країни користувачем у випадаючому списку. |
| `http://localhost/mycountries/v1.1/getcity/:cityname` | Повертає інформацію про місто країни на основі отриманого ідентифікатора (аліаса) `:cityname`.<br>_Приклад: `.../getcity/kyiv`_<br>У front-end частині не використовується, але може бути реалізовано. |

> *Примітка.* Дозволяється використовувати відмінні URL-адреси, але бажано все ж таки реалізувати вказаний вигляд.

**2. Інформація про країни зберігається в базі даних.**

**3. Технологія реалізації серверної частини — РНР**, але також може бути запропонована студентами самостійно (Java, Python, JS та ін).

**4. Дозволяється використовувати вебфреймворки** за умови ґрунтовного розуміння їх роботи.

**5. Формат даних API — JSON.** Необхідно реалізувати структуру сумісну з front-end частиною.

**6. Після розробки АРІ необхідно підключити запропоновану реалізацію до front-end частини лабораторної роботи № 4.**

                     +----------------------+

### Контрольні питання

1. Формати даних JSON та XML.
2. Принцип дії НТТР-сервера (Nginx, Apache).
3. Основні поняття реляційних баз даних (поле, ключ, відношення та ін.).
4. Архітектурний шаблон MVC.
5. Засоби РНР для роботи з БД.
